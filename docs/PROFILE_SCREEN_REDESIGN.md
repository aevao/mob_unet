# Переработка страницы профиля - Архитектура и дизайн

## Проблемы старого дизайна

1. **Плохая визуальная иерархия** - все элементы имели одинаковый вес
2. **Перегруженность** - слишком много информации на экране одновременно
3. **Аватар не является фокусной точкой** - маленький размер, не привлекает внимание
4. **Формальный список** - данные выглядели как форма, а не профиль
5. **Морально устаревший дизайн** - не соответствует современным стандартам

## Новое архитектурное решение

### 1. Компонентная структура

```
ProfileScreen (контейнер)
├── ProfileHeader (верхний блок)
│   ├── Крупный аватар (128x128)
│   ├── Имя пользователя (крупный шрифт)
│   ├── Роль и группа
│   └── Кнопка редактирования
├── ProfileInfoSection (секция информации)
│   ├── Заголовок секции
│   ├── ProfileField (основные поля)
│   ├── ProfileField (вторичные поля - скрыты)
│   └── Кнопка "Подробнее"
└── ProfileSkeleton (состояние загрузки)
```

### 2. Визуальная иерархия

#### Header (ProfileHeader)
- **Аватар**: 128x128px (было 96x96px) - увеличен на 33%
- **Имя**: text-2xl, font-bold (было text-xl) - более заметно
- **Роль**: вторичный текст, меньший размер
- **Кнопка редактирования**: в правом верхнем углу, всегда доступна

**Почему так лучше:**
- Аватар становится фокусной точкой экрана
- Имя пользователя сразу бросается в глаза
- Кнопка редактирования логично расположена рядом с профилем

#### Информация (ProfileInfoSection)
- **Карточки вместо списка**: каждая секция в отдельной карточке
- **Иконки в цветных контейнерах**: визуально привлекательнее
- **Разделители между полями**: четкая структура
- **Сворачиваемые секции**: второстепенная информация скрыта

**Почему так лучше:**
- Меньше визуального шума
- Легче сканировать информацию
- Фокус на важных данных (телефон, дата рождения)

### 3. Оптимизация контента

#### Основные поля (всегда видимы):
- Телефон
- Дата рождения

#### Вторичные поля (под "Подробнее"):
- Имя
- Фамилия
- Отчество
- Email
- Номер студенческого билета

**Почему так лучше:**
- Пользователь видит только важную информацию
- Меньше прокрутки
- Более чистое восприятие

### 4. UX улучшения

#### Skeleton Loading
- Показывает структуру экрана во время загрузки
- Пользователь понимает, что происходит
- Улучшает восприятие скорости загрузки

#### Пустое состояние аватара
- Красивая иконка вместо пустого места
- Понятно, что можно загрузить фото

#### Режим редактирования
- Кнопка редактирования всегда видна
- Плавный переход между режимами
- Кнопки сохранения появляются только при редактировании

### 5. Технические детали

#### Компоненты

**ProfileHeader.tsx**
```typescript
- Аватар с overlay для редактирования
- Крупное имя пользователя
- Роль и группа с иконками
- Кнопка редактирования в углу
```

**ProfileField.tsx**
```typescript
- Иконка в цветном контейнере
- Label и значение
- Поддержка редактирования
- Опциональная навигация (chevron)
```

**ProfileInfoSection.tsx**
```typescript
- Группировка полей
- Разделение на основные и вторичные
- Сворачиваемая секция "Подробнее"
- Разделители между полями
```

**ProfileSkeleton.tsx**
```typescript
- Skeleton для header
- Skeleton для полей
- Адаптация под light/dark mode
```

## Цветовая схема

### Light Mode
- Фон: белый
- Текст: #111827 (gray-900)
- Вторичный текст: #6B7280 (gray-500)
- Акцент: #2563EB (blue-600)
- Иконки в контейнерах: #EFF6FF (blue-50)

### Dark Mode
- Фон: темный
- Текст: #E5E7EB (gray-200)
- Вторичный текст: #9CA3AF (gray-400)
- Акцент: #60A5FA (blue-400)
- Иконки в контейнерах: rgba(59, 130, 246, 0.3) (blue-900/30)

## Типографика

- **Имя**: 24px, bold (text-2xl)
- **Роль**: 14px, regular (text-sm)
- **Label**: 12px, medium (text-xs)
- **Значение**: 16px, medium (text-base)
- **Заголовок секции**: 16px, semibold (text-base)

## Отступы и сетка

- Базовый отступ: 16px (px-4)
- Большой отступ: 24px (px-6)
- Вертикальные отступы: 16px, 24px, 32px
- Разделители: 1px

## Анимации

- Pressable: opacity-70 при нажатии
- Плавное раскрытие секции "Подробнее"
- Skeleton: статичный (можно добавить shimmer)

## Адаптивность

- Все размеры в относительных единицах
- Flexbox для адаптации
- Поддержка разных размеров экранов

## Заключение

Новый дизайн решает все проблемы старого:
- ✅ Улучшенная визуальная иерархия
- ✅ Минималистичный интерфейс
- ✅ Аватар как фокусная точка
- ✅ Удобное восприятие данных
- ✅ Современный внешний вид
- ✅ Отличная мобильная эргономика
